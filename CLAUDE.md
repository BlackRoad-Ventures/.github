# CLAUDE.md - AI Assistant Guidelines

This document provides guidance for AI assistants working with the BlackRoad-Ventures `.github` repository.

## Repository Overview

This is a **GitHub organization profile repository** (`.github`) for **BlackRoad-Ventures**, one of 15 specialized organizations in the BlackRoad ecosystem focused on investment, partnerships, and business development.

### Purpose

The `.github` repository serves special functions in GitHub:
- **Organization Profile**: The `profile/README.md` file is displayed on the organization's public GitHub page
- **Community Health Files**: Can contain organization-wide templates and guidelines
- **Default Files**: Files here serve as defaults for all repositories in the organization

## Development Guidelines

### When Modifying the Organization Profile (`profile/README.md`)

1. **Maintain Brand Consistency**: Keep the BlackRoad branding and ecosystem references
2. **Preserve Structure**: Keep the existing sections (Mission, Sister Organizations, Resources)
3. **Use Markdown Best Practices**: Proper headings, links, and formatting
4. **Test Links**: Ensure all organization and resource links are valid

### When Adding Community Health Files

Common files that can be added to this repository:
- `CODE_OF_CONDUCT.md` - Community conduct guidelines
- `CONTRIBUTING.md` - Contribution guidelines (applies to all org repos)
- `SECURITY.md` - Security policy
- `SUPPORT.md` - Support resources
- `FUNDING.yml` - Sponsorship configuration
- `ISSUE_TEMPLATE/` - Issue templates for the organization
- `PULL_REQUEST_TEMPLATE.md` - PR template for the organization

### Commit Message Convention

Use conventional commit messages:
- `feat:` - New features or content
- `fix:` - Bug fixes or corrections
- `docs:` - Documentation updates
- `chore:` - Maintenance tasks

Example: `feat: Add CODE_OF_CONDUCT.md for organization`

## AI Assistant Best Practices

### Do

- Keep changes minimal and focused
- Preserve existing branding and ecosystem references
- Validate all external links before adding
- Follow the established formatting style
- Consider the public-facing nature of profile content

### Don't

- Remove or alter ecosystem organization references without explicit request
- Add promotional or marketing content without approval
- Modify the footer attribution without guidance
- Create unnecessary files or restructure without clear purpose
